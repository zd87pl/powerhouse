# Railway Deployment Skill

> Deploy Powerhouse-scaffolded projects to Railway.app

## Prerequisites

```bash
# CLI is installed by bootstrap-clis.sh, or:
curl -fsSL https://railway.app/install.sh | sh

# Authenticate
railway login    # Opens browser for OAuth
```

## Quick Deploy

```bash
cd my-project
railway init              # Create project (one-time)
railway link              # Link existing project
railway up                # Deploy current directory
```

## Scaffoled Project Types

### Next.js Frontend

```bash
powerhouse scaffold my-store --stack nextjs --deploy railway
cd my-store

# Railway auto-detects next.config.js and creates:
# - Build command: npm run build
# - Start command: npm start
# - Output directory: .next

railway up
```

### FastAPI Backend

```bash
powerhouse scaffold my-api --stack fastapi --deploy railway
cd my-api

# Add railway.toml (generated by scaffold)
```toml
[deploy]
startCommand = "uvicorn main:app --host 0.0.0.0 --port $PORT"
runtime = "V2"
isStatic = false
```

railway up
```

### Full-Stack (Next.js + FastAPI + Postgres)

```bash
powerhouse scaffold my-app --stack fullstack --deploy railway
cd my-app

# Railway provisions automatically:
railway add --database postgres
railway add --database redis

# Environment variables auto-injected:
# DATABASE_URL, REDIS_URL, PORT

railway up
```

## Configuration Files

### railway.json (Recommended)

```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS",
    "buildCommand": "npm run build"
  },
  "deploy": {
    "startCommand": "npm start",
    "healthcheckPath": "/api/health",
    "sleepApplication": false,
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
```

### railway.toml (Alternative)

```toml
[build]
builder = "nixpacks"

[deploy]
startCommand = "uvicorn main:app --host 0.0.0.0 --port $PORT"
runtime = "V2"
```

## Environment Variables

```bash
# Set secrets
railway variables set DATABASE_URL="postgresql://..."
railway variables set OPENROUTER_API_KEY="sk-..."

# Bulk from .env file
railway variables --file .env

# View all
railway variables
```

## Local Development with Railway

```bash
# Connect to production DB locally
railway connect postgres

# Run with local tunnel
railway run npm run dev
```

## PR Preview Environments

Railway automatically deploys every PR:

```
main branch    →  https://my-app.up.railway.app
PR #42         →  https://my-app-pr-42.up.railway.app
```

Disable in project settings if not desired.

## Domain & SSL

```bash
# Custom domain
railway domain
# Generates: my-app.up.railway.app

# Add your own
railway domain add api.example.com
# Verify DNS, Railway handles SSL automatically
```

## Pricing Notes

| Tier | Best For |
|------|----------|
| **Hobby** ($5/mo + usage) | Side projects, MVPs |
| **Pro** ($20/mo + usage) | Production apps, teams |
| **Enterprise** | Custom SLA, dedicated support |

Usage billed by the second. No idle cost for **sleep-enabled** services.

## When to Choose Railway vs Fly.io vs Vercel

| | Railway | Fly.io | Vercel |
|---|---|---|---|
| **Always-on backend** | ✅ Managed, easy | ✅ Cheaper at scale | ⚠️ Serverless limits |
| **Auto-suspend idle** | ✅ (configurable) | ✅ Native | ✅ |
| **Edge deployment** | ❌ 6 regions | ✅ 30+ regions | ✅ Global CDN |
| **GPU** | ❌ | ❌ | ❌ |
| **Managed DB** | ✅ Postgres + Redis | ❌ (external) | ❌ |
| **PR previews** | ✅ Auto | ❌ Manual | ✅ |

**Rule of thumb:**
- Frontend-only → Vercel
- Backend + DB, small team → Railway
- Backend, cost-sensitive, edge → Fly.io
- GPU/training → RunPod

## Troubleshooting

| Issue | Fix |
|-------|-----|
| "No service found" | `railway init` first |
| Port binding | Always use `$PORT` env var |
| Build fails | Check `railway.json` buildCommand |
| DB connection | Use `DATABASE_URL` ( Railway injects) |
| Sleep timeout | Set `sleepApplication: false` in config |
